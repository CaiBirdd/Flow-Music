# Music Store 重构演练

本文档总结了对 Music Store (`store/music.ts`) 及相关组件进行的全面重命名重构，旨在提高代码的清晰度和可维护性。

## 🎯 目标

解决音乐播放逻辑中的命名歧义：

- `songs` (复数) 曾用于表示当前播放的一首单曲。
- `currentItem` (模糊) 曾用于表示当前*正在查看*的歌单。
- `runtimeList` (模糊) 曾用于表示播放队列。

## 🔄 变更内容

### 1. Store 状态与 Actions (`store/music.ts`)

| 旧名称                     | 新名称                         | 描述                        |
| :------------------------- | :----------------------------- | :-------------------------- |
| `state.songs`              | `state.currentSong`            | 当前正在播放的单曲。        |
| `state.currentItem`        | `state.viewingPlaylist`        | 当前主视图中打开的歌单。    |
| `state.runtimeList`        | `state.playQueue`              | 实际的播放队列列表。        |
| `action.updateCurrentItem` | `action.updateViewingPlaylist` | 更新当前查看歌单的 Action。 |
| `action.updateRuntimeList` | `action.updatePlayQueue`       | 更新播放队列的 Action。     |

### 2. API 类型 (`api/musicList.ts`)

| 旧名称        | 新名称          | 描述                             |
| :------------ | :-------------- | :------------------------------- |
| `CurrentItem` | `QueuePlaylist` | 包含歌曲列表的歌单结构类型定义。 |

### 3. 组件 Props

为了与 Store 的变更保持一致，以下组件的 Props 也进行了重命名：

**`SongList`, `MusicPlayer`, `DetailLeft`, `DetailRight`, `ProgressBar`**

| 旧 Prop | 新 Prop       |
| :------ | :------------ |
| `songs` | `currentSong` |

## 🛠️ 涉及文件

重构已应用于整个渲染器代码库，包括：

- **Views (视图)**: `UserCover`, `SearchList`, `PlayList`, `MusicCloud`, `LatelyPlay`, `DailyRecommend`
- **Components (组件)**: `SongList`, `SongInfo`, `MusicPlayer`, `MusicDetail`, `PlayListDrawer`
- **Hooks**: `usePlayList.ts`, `useMusic.ts`
- **Main App**: `App.vue`
