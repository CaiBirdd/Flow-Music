# Pinia 状态管理分析

本项目中使用了 `pinia` 作为状态管理，将不同领域的全局状态进行了清晰的划分。目前项目中定义了 4 个不同的 Store 模块：

## 1. 音乐播放核心数据 (\`useMusicAction\` -> \`store/music.ts\`)

这是项目中最核心的 Store (id: \`musicActionId\`)，几乎包含所有与播放器逻辑相关的数据。它不仅管理当前播放状态，还负责维护播放队列、搜索历史等，部分配置还通过 \`localStorage\` 的 \`MUSIC_CONFIG\` 进行持久化。

- \`musicUrl\`: 当前正在播放的音频链接 (MP3)。
- \`currentSong\`: 当前歌曲的详细信息对象（包含歌手、专辑、时长等）。
- \`viewingPlaylist\`: 用户当前选中准备查看的歌单列表数据（注意：不一定是正在播放的歌单）。
- \`playQueue\`: 当前实际正在播放的包含歌曲数组列表数据的播放队列。
- \`playQueueIds\`: 当前播放队列的所有歌曲 ID 数组（用于上一首/下一首的快速索引）。
- \`lyric\`: 经解析处理后的带时间轴的歌词数组（合并了原文和翻译）。
- \`currentTime\`: 音频当前的播放进度（单位：秒）。
- \`noTimestamp\`: 标记当前歌曲是否为纯音乐或无时间轴歌词。
- \`bgColor\`: 根据当前播放歌曲封面计算出的主题背景渐变色数组（如 \`["rgb(...)", "rgb(...)"]\`）。
- \`videoPlayUrl\`: 歌曲动态封面的相关视频 URL，如果有的话。
- \`orderStatusVal\`: 音乐播放模式：\`0\` 代表列表循环，\`1\` 代表随机播放，\`2\` 代表单曲循环。
- \`load\`: 用于标识音频是否正处于加载状态。
- \`index\`: 当前正在播放的歌曲在 \`playQueue\` 队列中的下标位置。
- \`lastIndexList\`: 随机播放模式下的历史播放索引栈，方便点击“上一首”时能按原路径回退。
- \`searchList\`: 临时存储从接口查询到的搜索结果列表。

## 2. 用户个性化设置 (\`useSettings\` -> \`store/settings.ts\`)

该 Store (id: \`settingsId\`) 主要用于管理用户的偏好与系统环境设置。它实现了**响应式的本地持久化**，在应用启动时会自动从 \`localStorage\` 中的 \`USER_SETTINGS\` 读取并恢复。

- \`baseUrl\`: 后端 API 接口请求的主地址（默认读取环境变量，支持随时动态修改并同步给 axios）。
- \`lyricBgMode\`: 歌词界面的背景模式（比如 \`rgb\` 常规模式或 \`rhythm\` 律动模式）。
- \`bold\`: 全局配置是否将特定字体加粗显示。
- \`font\`: 全局页面所使用的自定义字体系列（如 \`Avenir, Helvetica\` 等，修改后会直接操作 DOM 生效）。

## 3. 全局 UI 交互标志 (\`useFlags\` -> \`store/flags.ts\`)

用于解耦组件之间显示与隐藏的复杂状态通讯，管理当前应用的各级全局视图与弹窗状态 (id: \`flagsId\`)。

- \`isMaximize\` / \`isMinimize\`: 客户端窗口当前是否处于最大化或最小化状态。
- \`isOpenDetail\`: 音乐详情页（即歌词与大封面的全屏显示页）是否处于处于展开状态。
- \`isOpenSearch\`: 顶部的搜索框弹窗层是否被打开。
- \`isOpenDrawer\`: 位于页面右侧或边缘的播放列表抽屉是否被打开。
- \`isOpenLogin\`: 系统的全局登录弹窗是否显示。
- \`maxCount\`: 记录页面层级深浅，主要用于控制路由的“前进/返回”逻辑。

## 4. 动态背景主题色 (\`useTheme\` -> \`store/theme.ts\`)

专门用于辅助计算和渲染动态的视觉特效 (id: \`themeId\`)，特别是用于首页或全屏播放页根据封面动态改变大背景颜色的逻辑（依赖 \`colorthief\` 库）：

- \`bgColor\`: 存储最终经计算并压暗处理过的主题颜色（RGB 数组），用于渲染全局范围的页面背景。
- \`activeLayerIndex\`: 双缓冲切换层的逻辑指针（值为 \`0\` 或 \`1\`），它控制背景渐变在两个不同的 DIV 之间进行交替透明度淡入淡出，以实现颜色切换时的平滑过渡。
